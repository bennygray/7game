## 问题 1：JIT 上下文与 CLI `-p` 模式矛盾 ❌

**§1.1 说**："CLI `-p` 是无状态单次调用，每轮都要重新注入全部上下文"
**§三第三篇说**："Just-in-Time 上下文，编排器传文件路径引用，CLI Agent 按需读取"

**矛盾**：CLI `-p` 模式的 Agent **没有能力** "按需读取文件"。它不是交互式的，不能像 Claude Code 那样用 glob/grep 自主探索文件系统。它只能处理你一次性塞进去的 prompt。

**修正建议**：承认 JIT 上下文在 CLI `-p` 模式下**不可用**。我们能做的是：
- Python 编排器做 JIT（按阶段决定读哪些文件）
- **CLI 只接收编排器预组装好的上下文**（已经过筛选和压缩）
- 不要假设 CLI Agent 有自主探索能力

---

## 问题 2：M1 和 M1.5 边界模糊 ⚠️

**M1 说**："用单模型实现最简 Eval-Opt 循环"
**M1.5 说**："验证异构审查是否比同模型更有效"

**问题**：M1 的 Evaluator 也是 Claude（同模型）。这意味着 M1 的 "Evaluator" 本质上是 Claude 审查 Claude 自己的输出 — 这正是我们之前分析过的"橡皮图章"问题。

**疑问**：M1 如果只验证"骨架能跑"，那 Evaluator 质量好不好根本不重要。但这会不会浪费时间——M1 证明的只是 "subprocess + XML 解析能工作"，不需要一个真正的 Evaluator。

**修正建议**：
- M1 的目标应该更诚实：**它只验证技术管道（subprocess → stdin → stdout → XML 解析 → 文件写入）**
- 不需要假装有一个有意义的 Eval-Opt 循环
- 改名为 "M1: 技术管道验证"

---

## 问题 3：SGE 阶段 "编码" 的可行性疑问 ⚠️

**§2.3 说**：Coder (`claude -p` via stdin) 负责"编码实施"

**问题**：CLI `-p` 是非交互的，它**不能在项目中创建/修改文件**。它只能在 stdout 里输出文本。这意味着：
- Claude `-p` 可以输出代码差异或完整文件内容到 stdout
- 但**不能像交互模式那样直接写入 src/ 目录的文件**
- Python 编排器需要解析输出，自己执行文件写入操作

**修正建议**：
- 在 SGE 阶段明确说明：Claude 输出代码到 stdout → Python 解析 → Python 写文件
- 或者考虑：SGE 的 Generator 使用 Claude Code 交互模式（`claude` 不带 `-p`），通过 subprocess 的 stdin 逐步传入指令

## 问题 5：Codex CLI 命令格式未验证 ⚠️

**§2.3 说**：Reviewer 使用 `codex exec`

**问题**：我们还没有验证过 Codex CLI 的实际命令格式。它的 stdin 管道支持情况、输出格式、错误处理方式都是未知的。M0 中虽然列了 `codex exec "hello"` 的检查，但没有针对 Codex 的 stdin 管道验证。

**修正建议**：
- M0 增加 Codex stdin 管道验证（如 `echo "review this" | codex exec`）
- 研究 Codex CLI 的文档确认准确的命令格式

## 问题 6：.reviews/ 在 .gitignore 中吗？ ⚡

**§2.5 和 §八都提到** `.reviews/` 是审查记录目录。

**未定义**：
- `.reviews/` 应该被 Git 管理（审计追溯）还是 `.gitignore`（临时文件）？
- `_prompt.md` 临时文件应该被忽略
- 如果 Git 管理，提交策略是什么？每次审查后自动 commit？

**修正建议**：明确定义 `.reviews/` 的 Git 策略。建议：
- `.reviews/*.md` → Git 管理（审计追溯）
- `.reviews/_prompt.md` → `.gitignore`（临时文件）

## 问题 7：缺少失败后的降级方案 ⚡

**M1.5 说**："如果 0 个差异 → 重新评估系统价值，考虑 STOP"

**问题**：如果 M1.5 结论是 STOP，我们还有别的选择吗？完全放弃？

**修正建议**：STOP 不应该是唯一选项。可能的降级方案：
- **降级 A**：只用同模型审查（虽然不够对抗，但有 Compaction + 结构化 prompt 改进后可能够用）
- **降级 B**：改为手动运行 Codex（不自动化，但人类可以按需跑）
- **降级 C**：等更好的模型/工具出现后再重启
