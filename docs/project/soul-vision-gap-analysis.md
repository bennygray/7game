# 🌍 远景全景图：有灵魂的 AI NPC 生活的世界

> **目标**：让每个弟子不是执行脚本的傀儡，而是拥有**自洽动机、差异化反应、可被理解的内心世界**的活灵魂。
> **评估日期**：2026-03-29 | **已完成 Phase**: A → B-α → C → D → E

---

## 全景架构：五层灵魂循环

一个真正「有灵魂」的 NPC 需要**五层系统的闭环**：

```
                    ┌─────────────────────────────────────────────┐
                    │              ⑤ 表现层 (Presentation)         │
                    │   玩家如何「看到」和「感知到」灵魂存在         │
                    └──────────────┬──────────────────────────────┘
                                   ↑ 输出
                    ┌──────────────┴──────────────────────────────┐
                    │              ④ 社交层 (Social)               │
                    │   NPC 之间的关系动态 + 群体涌现行为            │
                    └──────────────┬──────────────────────────────┘
                                   ↑ 社交结果
                    ┌──────────────┴──────────────────────────────┐
                    │              ③ 决策层 (Decision)             │
                    │   NPC 为什么选择做这件事而不是那件事           │
                    └──────────────┬──────────────────────────────┘
                                   ↑ 决策依据
                    ┌──────────────┴──────────────────────────────┐
                    │              ② 感知层 (Perception)           │
                    │   NPC 对事件的情绪反应 + 内心评价              │
                    └──────────────┬──────────────────────────────┘
                                   ↑ 感知来源
                    ┌──────────────┴──────────────────────────────┐
                    │              ① 数据层 (Identity)             │
                    │   NPC 是谁 — 道德/特性/记忆/关系              │
                    └─────────────────────────────────────────────┘
```

**闭环**：数据层定义身份 → 感知层产生情绪 → 决策层驱动行为 → 社交层改变关系 → 表现层被玩家看到 → 行为结果产生新事件 → 回到感知层。

## 逐层 Gap 分析

### ① 数据层 — NPC 是谁

> NPC 的「身份证」：性格特征、道德信仰、与他人的关系、过往记忆。

| 子系统 | 状态 | 实现位置 | 缺失/问题 |
|--------|:----:|----------|----------|
| 五维性格（aggressive/persistent/kind/lazy/smart） | ✅ | `game-state.ts` PersonalityTraits | 生成时固定，永不变化 |
| 道德双轴（goodEvil/lawChaos） | ✅ | `soul.ts` MoralAlignment | 漂移速度极微（0.001/tick），几乎无感 |
| 先天特性（14个 TraitDef） | ✅ | `trait-registry.ts` | 数据已定义 |
| 后天特性触发 | 🔶 | `soul-engine.ts` checkAcquiredTraits | 只有 2 个后天特性，触发所需计数在**内存中**不持久化 |
| 关系边（affinity/tags） | ✅ | `game-state.ts` RelationshipEdge | 数据就绪 |
| 短期记忆（最近情绪/事件） | ❌ | — | **无**。NPC 没有记忆昨天发生了什么 |
| 长期记忆摘要 | ❌ | — | **无**。AI 无法回忆重大事件 |
| 个人目标/欲望 | ❌ | — | **无**。NPC 没有「想要」什么 |

**层评估**：🔶 **半成品** — 身份数据已有框架，但缺少记忆和目标，NPC 是一个有性格标签但没有阅历的人。

---

### ② 感知层 — NPC 对事件的反应

> 事件发生后，NPC 产生情绪 + 内心独白 + 关系变化。

| 子系统 | 状态 | 实现位置 | 缺失/问题 |
|--------|:----:|----------|----------|
| 事件总线（EventBus） | ✅ | `event-bus.ts` | 7 种事件类型已覆盖 |
| 候选情绪池（7×2=14组） | ✅ | `emotion-pool.ts` | 规则引擎预筛 |
| AI 情绪评估（结构化 JSON） | 🔶 | `soul-prompt-builder.ts` | Prompt 已就绪，**但 AI 路径未接入**，仅 fallback |
| Fallback 规则引擎 | ✅ | `soul-engine.ts` fallbackEvaluate | 随机选情绪 + 模板独白 |
| Delta 方向修正 + 夹值 | ✅ | `emotion-pool.ts` | correctDeltaDirection + clampDelta |
| 特性→情绪权重（emotion-weight） | ✅ | `soul-engine.ts` applyTraitWeights | 正负权重均已处理 |
| 关系标签→Delta 修正（R-E13） | ❌ | — | PRD 已定义 `friend→×1.2`、`rival→×1.3` 等，**但代码完全未实现** |
| 情绪持续/记忆 | ❌ | — | 情绪是瞬时的，下一秒就消失了。没有「余怒未消」 |

**层评估**：🔶 **半成品** — 事件→情绪管线运转正常，但 AI 路径断开、关系标签效果未生效、无情绪持续。

---

### ③ 决策层 — NPC 为什么选择这个行为 ⚠️ 最大缺口

> NPC 选择做什么、怎么做，基于性格/特性/情绪/关系。

| 子系统 | 状态 | 实现位置 | 缺失/问题 |
|--------|:----:|----------|----------|
| 五维性格→行为权重 | ✅ | `behavior-tree.ts` getPersonalityWeights | 基础运转 |
| **特性→行为权重（behavior-weight）** | ❌ | — | `trait-registry.ts` 定义了 explore+0.3、alchemy+0.5 等，**但 `getPersonalityWeights` 完全不读 `traits`** |
| **关系标签→行为权重（R-E13）** | ❌ | — | PRD 定义了 `friend→合作×1.3`、`admirer→meditate×1.1`，**完全未实现** |
| **情绪→行为偏移** | ❌ | — | 愤怒的弟子应该更倾向 bounty/explore，悲伤的应该倾向 rest，**完全未实现** |
| 道德→行为约束 | ❌ | — | 善良的弟子是否更倾向 farm/alchemy？**无任何连接** |
| 弟子间协作行为 | ❌ | — | 朋友是否更倾向合作任务？**没有合作行为的概念** |
| 目标驱动行为 | ❌ | — | NPC 不因「想突破」而更频繁修炼 |

**层评估**：❌ **严重断裂** — **这是整个灵魂循环中最大的断裂点**。灵魂系统产生的 traits/emotion/affinity/moral **全部是死数据**，不影响弟子的实际行为选择。弟子的行为仍然 100% 由生成时的五维性格决定。

## 断裂点优先级排序

> [!IMPORTANT]
> **最致命的问题不是缺什么功能，而是灵魂系统产生的所有数据（traits/emotion/affinity/moral）都是死数据——不影响弟子的行为选择。**

| 优先级 | 断裂点 | 影响 | 涉及层 |
|:------:|--------|------|:------:|
| **P0** | 特性 behavior-weight 不影响行为决策 | 14个特性的 behavior-weight 全是死数据 | ③ |
| **P0** | 关系标签效果没有接入任何系统 | friend/rival 标签只存不用 | ②③④ |
| **P1** | 情绪不影响行为 | 弟子愤怒 vs 喜悦选择相同行为 | ②③ |
| **P1** | AI 路径未接入 | soul-prompt-builder 是死代码 | ② |
| **P2** | 对话内容不读取关系 | 好友和仇人说话完全一样 | ④⑤ |
| **P2** | 无情绪持续/记忆 | 上一秒愤怒，下一秒无感 | ①② |
| **P2** | 弟子档案缺失 | 玩家看不到特性/道德/关系 | ⑤ |
| **P3** | 高级标签(mentor/admirer/grudge)未实现 | 社交关系层次单一 | ④ |
| **P3** | 道德趋同（亲密弟子互影响）未实现 | 社交涌现不存在 | ④ |
| **P3** | 个人目标/欲望 | NPC 缺少内在驱动力 | ① |

## 建议推进路径

### 🎯 近期（做完后灵魂系统「闭环」—— 约 3~5 天）

**目标**：让特性/情绪/关系真正驱动行为，闭合③决策层断裂

1. **行为权重叠加**：`getPersonalityWeights` 读取 `traits[].effects` 中 `behavior-weight` 类型的效果叠加
2. **关系标签效果接入**：`friend/rival` 的 delta 倍率（×1.2/×0.8 等）接入 soul-engine
3. **当前情绪→行为偏移**：弟子最近一次强烈情绪暂时调节行为权重
4. **AI 路径接入**：soul-prompt-builder 连接 llm-adapter

### 🔮 中期（灵魂可被「观察」—— 约 3~5 天）

**目标**：让玩家能看到、理解、感受灵魂存在

5. 弟子档案面板（特性/道德/关系总览）
6. 关键灵魂事件高亮/置顶
7. 对话系统读取 affinity（好友/仇人不同语气）
8. 短期情绪持续（最近情绪保留 N tick）

### 🌟 远期（灵魂产生「涌现」—— 无限期）

**目标**：NPC 自发产生超预期的社交动态

9. 高级标签（mentor/admirer/grudge）
10. 社交事件（争吵/和好/结拜）
11. 道德趋同（亲密弟子互相影响）
12. 个人目标/欲望系统
13. 弟子叛逃/弟子推举事件
