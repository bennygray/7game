# 🌍 远景全景图：有灵魂的 AI NPC 生活的世界

> **目标**：让每个弟子不是执行脚本的傀儡，而是拥有**自洽动机、差异化反应、可被理解的内心世界**的活灵魂。
> **评估日期**：2026-03-29 | **已完成 Phase**: A → B-α → C → D → E
> **最后更新**：2026-03-29 V3（基于世界线推演定案重构）

## 全景架构：六层灵魂循环（V3 新增第⑥层）

一个真正「有灵魂」的 NPC 需要**六层系统的闭环**：

```
                    ┌─────────────────────────────────────────────┐
                    │              ⑥ 世界层 (World)               │
                    │   NPC 在哪里、世界发生了什么、谁遇到了谁      │
                    └──────────────┬──────────────────────────────┘
                                   ↑ 事件源
                    ┌──────────────┴──────────────────────────────┐
                    │              ⑤ 表现层 (Presentation)         │
                    │   玩家如何「看到」和「感知到」灵魂存在         │
                    └──────────────┬──────────────────────────────┘
                                   ↑ 输出
                    ┌──────────────┴──────────────────────────────┐
                    │              ④ 社交层 (Social)               │
                    │   NPC 之间的关系动态 + 群体涌现行为            │
                    └──────────────┬──────────────────────────────┘
                                   ↑ 社交结果
                    ┌──────────────┴──────────────────────────────┐
                    │              ③ 决策层 (Decision)             │
                    │   NPC 为什么选择做这件事而不是那件事           │
                    └──────────────┬──────────────────────────────┘
                                   ↑ 决策依据
                    ┌──────────────┴──────────────────────────────┐
                    │              ② 感知层 (Perception)           │
                    │   NPC 对事件的情绪反应 + 内心评价              │
                    └──────────────┬──────────────────────────────┘
                                   ↑ 感知来源
                    ┌──────────────┴──────────────────────────────┐
                    │              ① 数据层 (Identity)             │
                    │   NPC 是谁 — 道德/特性/记忆/关系              │
                    └─────────────────────────────────────────────┘
```

**V3 关键变更**：新增第⑥层"世界层"。这是之前旧 Gap 分析中**完全忽略的维度**——没有世界层，②~⑤的一切都无从验证。

**闭环**：数据层定义身份 → ⑥**世界层产生事件** → 感知层产生情绪 → 决策层驱动行为 → 社交层改变关系 → 表现层被玩家看到 → 行为改变世界 → 回到世界层。

## 逐层 Gap 分析（V3）

### ⑥ 世界层 — NPC 在哪里、世界发生了什么 `0%` ⚠️ 根本性缺失

> [!CAUTION]
> **这是旧版 Gap 分析完全遗漏的维度，也是整个系统最大的盲区。**
> 没有世界层，灵魂系统产生的所有数据都无法被验证——连一个"被偷丹药后质问"的场景都无法在游戏中自然产生。

| 子系统 | 状态 | 缺失/问题 |
|--------|:----:|----------|
| 空间/地点系统 | ❌ | 弟子不在任何地方。EXPLORE 只是标签，不去任何地方 |
| NPC 碰面机制 | ❌ | 8 个弟子各自独立运行，互不知道对方存在 |
| 世界事件池 | ❌ | 不存在"天降灵器""妖兽来袭"等随机事件 |
| 因果关系事件 | ❌ | 弟子行为不产生任何影响他人的后果 |
| 事件分级系统 | ❌ | 所有事件都是同一级别的流水账 |
| Storyteller 节奏控制 | ❌ | 无戏剧节奏调控 |
| 宗门道风立场 | ❌ | 宗门无道德/气质属性，只有 reputation |
| T2 幕后 NPC | ❌ | 世界只有宗内 8 个弟子，没有外部角色 |
| 离线结算 | ❌ | 引擎停止时世界完全静止 |
| 玩家干预（实时+事后） | ❌ | 玩家无法干预任何弟子行为 |

**层评估**：❌ **完全缺失** — 这不是"功能不完善"，而是"这一整层都不存在"。

## 断裂点优先级排序（V3 重排）

> [!IMPORTANT]
> **最致命的问题不再只是"灵魂数据是死数据"——更根本的是：世界里根本不产生需要灵魂来处理的事件。**
> 世界层（⑥）的缺失使得 ②~⑤ 的一切改进都无法被验证。

| 优先级 | 断裂点 | 影响 | 涉及层 |
|:------:|--------|------|:------:|
| **P0** | **世界层完全缺失** | 无空间、无碰面、无世界事件、无事件分级 → 一切后续无从验证 | ⑥ |
| **P0** | 宗门无道风立场 | 无法区分正/邪/仁/霸的游戏体验 | ⑥① |
| **P1** | 特性 behavior-weight 不影响行为 | 14个特性死数据 | ③ |
| **P1** | 关系标签效果未接入 | friend/rival 只存不用 | ②③④ |
| **P1** | 情绪不影响行为 | 愤怒 vs 喜悦行为一样 | ②③ |
| **P1** | AI 路径未接入生产 | soul-prompt-builder 是死代码 | ② |
| **P2** | 无玩家交互（干预/裁决） | 玩家只能看不能做 | ⑤ |
| **P2** | 对话不读关系 | 好友仇人说话一样 | ④⑤ |
| **P2** | 无事件分级显示 | 所有日志同一级别 | ⑤ |
| **P3** | 高级标签/社交事件/道德趋同 | 社交涌现不存在 | ④ |
| **P3** | 记忆/目标/自驱 | NPC 缺少内在驱动力 | ① |

## AI 容量评估摘要

> GPU 模式下（`-ngl 99`），0.8B 模型吞吐 ~12,000 次/小时。
> T1 可见 NPC 仅消耗 ~18 次/小时（0.15%）。
> 剩余 99.85% 可用于 T2 幕后 NPC。
> **AI 算力不是瓶颈。**
