# 🌍 SOUL VISION 重思记录 (3)：混合决策引擎架构推演（约束资源池方案）

> **记录时间**：2026-03-29
> **说明**：基于“0.8B 模型在约束资源池下表现可控”的重要测试共识，本文档正式推演如何彻底融合被割裂的 Phase F 与 Phase G，提出一套兼顾“活人感”与“算力边界”的全新灵魂决策架构。本文档不包含直接代码修改。

## 1. 废弃原 Roadmap 的“灵肉割裂”路线

首先，我们必须在战略上彻底终结原 `SOUL-VISION-ROADMAP.md` 中死板的切割设计：
- ❌ **废弃原 Phase F**：不再使用极其庞大、难以维护和缺乏涌现感的“数学权重树”来单独决定 NPC 的行为走向。
- ❌ **废弃原 Phase G**：不再让 AI 仅仅沦为“根据干瘪的战斗结算画面写战后感言”的廉价剧本生成器。

我们要走向的是：**让 AI 拥有“选择权”，让引擎掌握“执行权”。** 

## 2. 破局之道：The "Constrained Pool" 混合决策流

既然我们明确了 0.8B 模型在给予特定资源池（候选列表）时，其归类和选择能力是合格的，那么全新的核心架构应当是一套**“引擎提名 -> AI 拍板 -> 引擎执行”的闭环 Pipeline**。

### 核心运作机制：

#### 第一步：引擎层 —— 提名阶段（Candidate Generation）
当轮到某位 NPC 决策行动时（Tick 触发），深层的系统引擎不再直接扔骰子算概率，而是根据该 NPC 当前的处境、坐标、境界、包裹，**合法化地生成一个“行为候选池（Action Pool）”**。
*   *例如李清风当前的候选池*：`[A. 闭关修炼(Meditate), B. 探索历练(Explore), C. 炼制引气丹(Alchemy), D. 找张三寻仇(Attack:张三)]`

#### 第二步：组装层 —— 灵魂注入（Soul Prompting）
`soul-prompt-builder` 登场。它将该 NPC 的所有灵魂数据打包：
*   【身份】胆魄如虹、善嫉、星河剑派
*   【情绪】刚才张三当众羞辱了他，当前情绪“暴怒”
*   【候选行为池】（把第一步生成的 A, B, C, D 选项喂入）
*   **【约束指令】**：“作为李清风，请严格从 [A, B, C, D] 中选择你接下来最想做的一件事，并用一句话说出你的内心独白。格式必须包含 `<Decision:选项>`。”

#### 第三步：大模型层 —— 决策与表达（AI Inference）
哪怕是 0.8B 模型，在这个极端约束的 Prompt 下也能稳定输出：
*   `"张三竖子，欺人太甚！我定要让他血债血偿！ <Decision:D>"`

#### 第四步：解析执行层 —— 降维打击（Regex Parsing & Execution）
系统截获这句回答。
1. 将“张三竖子...”这句带有强烈性格色彩的话记录到 MUD 系统作为**灵魂日志**（对外表现，相当于替换掉了寻找车轱辘话的过程）。
2. 使用正则表达式精准提取 `<Decision:D>`。
3. 引擎拿着 `D`（Attack:张三），调用底层的硬代码战斗逻辑开始结算伤害。如果模型偶尔发癫生成了 `<Decision:E>`，引擎触发 Fallback 机制（比如降级为闭关）。

---

## 3. 这套新架构的巨大优势

1. **绝对可控的边界**：AI 永远不可能生成导致游戏崩溃的指令（比如凭空虚构出一个没有的法宝去卖），因为选项池（Pool）是底层引擎生成的。
2. **极强的涌现与活人感**：NPC 的行为选择会因为他上一个 Tick 结下的仇怨（情绪）、他的先天性格，而在候选池中发生动态偏移。这就是你渴望的**自驱力**。
3. **彻底释放 0.8B 潜力**：不要求它算数，不要求它凭空造 JSON，只要求它“根据人设，做选择题并配音”。这是小模型性价比最高的杀手锏。

## 结论与下一步：
如果这个“混合约束决策池（引擎给菜单 -> AI 点菜 -> 引擎上菜）”的重思方向没有偏离您的愿景，那么先前的 Phase F (权重概率树) 和 Phase G (文本翻译机) 就失去了存在的意义，应当被直接废弃重构。这是极关键的一步共识。
