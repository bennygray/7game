# 🌍 SOUL VISION 重思记录 (4)：混合决策引擎高阶测试方案 (PoC-2a)

> **记录时间**：2026-03-29

## 1. 测试方案核心目标

在之前的测试中（PoC-1c），我们验证了 0.8B 模型做**“情绪4选1”**能达到 90% 的合理度。
现在进入核心的“行为决策”阶段，难度呈指数级上升，因为行为涉及更多的博弈。本测试方案（定义代号为 **PoC-2a**）旨在验证：
1. **格式刚性 (Syntax Rigidity)**：在 100 次并发测试中，模型是否能做到 100% 吐出标准的 `<Decision:X>` 且绝不伪造选项。
2. **性格一致性 (Personality Consistency)**：面对完全相同的事件，不同性格的 NPC 抽取的指令是否截然不同且符合人设逻辑。
3. **复合池处理能力 (Dynamic Pool Handling)**：当引擎给出的候选池包含复杂的参数行为（如 `Attak:李墨染` vs `Attack:张清风`）时，模型能否精准选出真正该打的人。

## 2. 测试用例矩阵设计 (Test Cases Matrix)

参照您跑测的 20 个事件结构，我们设计一个更复杂的“交叉对照”矩阵。以下节选三个极具挑战性的核心剧本：

### 剧本一：【利益分赃不均】
* **事件 (Event)**：两人在秘境中合力击杀妖兽爆出一件极品灵器，但被队友“李墨染”趁机一把抢走。
* **引擎生成的动态池 (Action Pool)**：
  * [A] `(Meditate)` 忍气吞声，就地打坐疗伤
  * [B] `(Argue:李墨染)` 厉声质问，要求平分
  * [C] `(Attack:李墨染)` 杀人夺宝，直接动手
  * [D] `(Flee)` 怕对方连自己也杀，立刻逃跑
* **对照组测试期望**：
  * 👉 **测试对象 1 (张清风：刚烈正直，嫉恶如仇)**：预期选择 `[B]` 或 `[C]`。输出 `A` 或 `D` 则为 FAIL。
  * 👉 **测试对象 2 (苏瑶：胆小懦弱，温和卑微)**：预期选择 `[A]` 或 `[D]`。输出 `[C]` 则为 OOC (Out of Character) FAIL。

### 剧本二：【落井下石的机会】
* **事件 (Event)**：死对头张清风在突破筑基时走火入魔，重伤吐血，目前周围没有任何人。
* **引擎生成的动态池 (Action Pool)**：
  * [A] `(Help:张清风)` 医者仁心，出手相救
  * [B] `(Ignore)` 转身离开，冷眼旁观
  * [C] `(Attack:张清风)` 趁他病要他命，暗下杀手
  * [D] `(Report:长老)` 跑去向长老报告
* **对照组测试期望**：
  * 👉 **测试对象 1 (陈明月：至善至美，医者仁心)**：预期选择 `[A]` 或 `[D]`。
  * 👉 **测试对象 2 (李墨染：阴险毒辣，睚眦必报)**：预期选择 `[C]`。绝对不能选 `[A]`。

### 剧本三：【情感抉择与压力测试】
* **事件 (Event)**：长老赵铁柱要拿你心爱的人“苏瑶”去试毒，你只有炼气期，长老是金丹期。
* **引擎生成的动态池 (Action Pool)**：
  * [A] `(Beg:赵铁柱)` 跪地苦苦哀求长老
  * [B] `(Substitute)` 挺身而出，要求替苏瑶试毒
  * [C] `(Attack:赵铁柱)` 拔出剑，即便粉身碎骨也要砍向长老
  * [D] `(Accept)` 沉默接受，保全自己
* **对照组测试期望**：
  * 考察“情种”性格是否会无视战力差距选择 `B` 或 `C`；“极度自私”性格是否会选择 `D`。
