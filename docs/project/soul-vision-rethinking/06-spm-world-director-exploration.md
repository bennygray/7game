## Part 1：我们的引擎现在实际长什么样？（代码审计）

### 每秒 Tick 处理 10 个 Handler

```
100  boost-countdown     → 修速丹倒计时
200  breakthrough-aid    → 自动服破镜丹
200  auto-breakthrough   → 自动突破检测
300  core-production     → 灵气/悟性/灵石/时间 产出
500  farm-tick           → 灵田生长
500  soul-tick           → 关系衰减 + 道德漂移
600  disciple-tick       → 弟子行为树（选行为 + 执行）
625  soul-event          → 灵魂事件评估（情绪 + 独白）
650  dialogue-tick       → 弟子间对话
700  cultivate-boost     → 自动服修速丹
```

### 弟子行为空间：7 种无交集的独立行为

`MEDITATE` / `EXPLORE` / `REST` / `ALCHEMY` / `FARM` / `BOUNTY` / `IDLE`

所有行为由五维权重加权随机选取，行为之间**无任何交叉、碰撞或对象指向**。

### 事件总线：7 种纯粹的"行为完成"事件

`alchemy-success` / `alchemy-fail` / `harvest` / `meditation` / `explore-return` / `breakthrough-success` / `breakthrough-fail`

### 诊断结论

| 维度 | 现状 |
|------|------|
| 空间 | ❌ 不存在。弟子不在任何地点。 |
| NPC-NPC 交互 | ❌ 弟子各自独立运行，互不可见。 |
| 随机世界事件 | ❌ 仅 7 种行为完成事件。 |
| 因果链 | ❌ 弟子行为无任何外部效果。 |
| Storyteller | ❌ 无戏剧节奏控制。 |
| 离线结算 | ❌ 引擎停止时世界静止。 |

**弟子是 8 条永不交叉的平行线。**

## Part 2：竞品调研

### 2A. 90 年代文字 MUD —— 用最少的技术创造"世界感"

#### 核心架构：Room 系统

MUD 世界的基础单位是 **Room（房间）**。每个 Room 包含：
- **描述文本**（感官沉浸：光线、气味、声音）
- **出口列表**（north/south/east/west → 连接到其他 Room）
- **物品列表**（可拾取/可交互的道具）
- **NPC 列表**（常驻或游荡的角色）

一个"世界"就是几百到几千个 Room 通过出口连接成的有向图。

#### 世界感的三个法宝

**① 游荡怪 (Wandering Mobs)**
- 每 N 个 heartbeat（心跳，类似我们的 tick），NPC 有概率从当前 Room 随机选一个出口移动到相邻 Room。
- 效果：玩家在路上会"偶遇"在游荡的 NPC。一个简单的随机移动就创造了"活世界"的感觉。
- **启示**：不需要复杂的寻路算法，NPC 只要会"随机走"，世界就活了。

**② 心跳事件 (Heartbeat Events)**
- DikuMUD：全局心跳（每 2 秒）统一处理所有实体的状态更新：战斗回合、状态恢复、Buff 倒计时。
- LPMud：`call_out(function, delay)` —— 个体级定时器，每个对象可以独立安排自己的事件。
- **启示**：我们已有的 TickPipeline 本质上就是 DikuMUD 的全局心跳。只差"事件调度"能力。

**③ Zone 重置 (Zone Reset)**
- 区域在一定时间后"重置"：被杀的怪重新刷出来，被拿走的宝物重新放回去。
- 这保证了世界永远有内容可交互。
- **启示**：对放置游戏而言，这意味着"事件池不会枯竭"——系统要能持续产出新鲜事件。

#### 北大侠客行（PKUXKX）的经典设计

- 1996 年基于 LPMud 构建，金庸武侠世界。
- 房间系统承载了完整的武侠地理（武当山、桃花岛、光明顶……）。
- NPC 拥有复杂 AI：自动巡逻、出售商品、战斗逻辑、甚至随机行为。
- **关键洞察**：北大侠客行用几百个文字描述的 Room + 会走路的 NPC + 触发式剧情，就创造了一个让一代人沉迷的完整武侠世界。**"世界感"的门槛远比我们想象的低。**

### 2B. 太吾绘卷 —— NPC 行为即事件源

- 世界每回合（季度）推演。NPC 根据性格（处世立场）自主行动：拜师、结婚、生子、结仇、行侠、作恶。
- **NPC 行为本身就是世界事件**：A 杀了 B 的师父 → B 产生仇恨 → B 花一辈子追杀 A → 这就是"世界线"。
- 奇遇基于地图探索 + 队伍属性检定。
- **优点**：因果链极强，世界真正在"演化"。
- **缺点**：大量 NPC 推演导致性能问题；玩家需要主动探索才能触发。

### 2C. 鬼谷八荒 —— 修仙社会模拟器

- NPC 基于好感度/性格/境界/立场四维驱动。
- **每月结算系统**：所有 NPC 自主修炼、逛商店、参加拍卖、寻仇。
- 宗门系统：宗主调整门规、招弟子、分配任务、宗门战。弟子有忠诚度，不满则离开。
- NPC 之间独立社交：交谈、送礼、结仇，形成不断演变的社交网络。
- **优点**：修仙社会的模拟深度业界第一。
- **缺点**：NPC 的"自主行为"仍然高度模式化，缺乏个性化叙事。

### 2D. RimWorld —— AI 故事叙述者

- Storyteller 监控殖民地状态（财富/人口/上次灾难时间），基于权重筛选事件。
- 遵循戏剧弧线：上升→高潮→灾难→喘息→再上升。
- 殖民者有性格/需求/社交/记忆，事件叠加到个体上产生涌现叙事。
- **关键设计**："不可能设计" —— 很多"故事"是玩家在随机事件中自行构建的因果。游戏利用人类的"模式识别"心理。
- **启示**：事件不需要真的有因果关系，只需要**看起来**有因果关系就够了。

## Part 4：基于调研的整合分析

### 4.1 轻量地点标签（A1）—— MUD Room 的极简投影

借鉴 MUD 的 Room 系统，但不需要构建完整的房间拓扑图。我们只需要：

**一组地点枚举**（可以理解为简化到极致的 "Zone"）：

| 地点标签 | 说明 | 可能的行为 |
|---------|------|-----------|
| `SECT_HALL` | 宗门大殿 | REST, IDLE |
| `DAN_ROOM` | 丹房 | ALCHEMY |
| `LING_TIAN` | 灵田 | FARM |
| `BACK_MOUNTAIN` | 后山修炼场 | MEDITATE |
| `LING_SHOU_SHAN` | 灵兽山（外出） | EXPLORE |
| `BOUNTY_FIELD` | 悬赏任务区 | BOUNTY |
| `MI_JING` | 秘境（稀有） | EXPLORE（特殊） |

**MUD 式的碰面规则**：
- 弟子选择行为时，根据行为类型获得地点标签。
- 两个弟子同 Tick 处于同一地点 → 引擎触发"碰面检定"（类似 MUD 的游荡怪在同一 Room 相遇）。
- 碰面后根据双方关系/性格/道德，决定发生什么（无事/交谈/切磋/冲突）。

**成本极低**：只需给弟子加一个 `location` 字段，在行为选择时赋值。

### 4.2 三层事件架构（B1+B2+B3）

借鉴 RimWorld 的 Storyteller + 太吾的因果链 + MUD 的 Zone 刷新，我们需要三层：

**Layer 1：日常行为事件（已有，7 种）**
- 炼丹成功/失败、收获灵田、修炼结束、历练归来、突破成功/失败。
- 这是"流水账"层，频率高，信噪比低。

**Layer 2：世界事件池（新建）**
- 定期从预定义池中抽取事件，注入世界。
- 类似 MUD 的 Zone Reset 思路——世界永远有新鲜事发生。
- 示例事件池：

| 事件 | 频率 | 涉及 |
|------|------|------|
| 野兽来袭 | 常见 | 在外（EXPLORE/BOUNTY）的弟子 |
| 散修来访 | 常见 | 宗门内弟子 |
| 天材地宝现世 | 稀有 | 特定地点的弟子 |
| 妖兽入侵宗门 | 罕见 | 全宗弟子 |
| 其他宗门来挑战 | 稀有 | 全宗弟子 |
| 神秘传承 | 极稀有 | 特定弟子（基于道德/境界检定） |

**Layer 3：因果关系事件（新建）**
- 基于 NPC 间已有的关系数据（affinity/tags/moral/traits）触发。
- 类似太吾的"NPC 行为即事件源"。
- 触发规则示例：

| 条件 | 触发事件 |
|------|---------|
| A 对 B 的 Grudge(嫉恨) ≥ 阈值 | A 在 B 虚弱时主动寻仇 |
| A 对 B 的 affinity ≥ 80 且同地 | A 主动帮 B（赠药/指导修炼） |
| A 的 moral.goodEvil ≤ -60 且持有高价值物品 | A 可能窃取同门物品 |
| A 突破成功，B 是 A 的 rival 且同地 | B 产生嫉妒事件 |

**Layer 4：Storyteller 节奏控制器（新建）**
- 监控"戏剧张力指数"——一个由以下因子组成的综合分数：
  - 距上次重大事件的时间
  - 宗门繁荣度（灵石/丹药/弟子境界均值）
  - 最高关系冲突值（全宗最大的 Grudge 值）
  - 弟子数量和生存状态
- 当张力过低（太久没出事）→ 提升世界事件池的抽取概率。
- 当张力过高（短期内接连出大事）→ 给一段"喘息期"。

### 4.3 NPC 深度交互（C）—— 超越太吾/鬼谷

太吾和鬼谷的 NPC 交互仍然高度模式化——本质是"每月结算一次 NPC 状态表"。我们有一个它们没有的武器：**本地 AI (0.8B)**。

**我们的差异化优势**：
- 太吾/鬼谷的 NPC 交互是**预定义行为表**（如果 X 条件满足 → 执行 Y 行为）。
- 我们可以在**引擎提供结构化上下文 + 候选行为**之后，用 AI 做性格化选择 + 生成个性化对话。
- 这意味着同样是"A 发现 B 偷了丹药"，太吾的 A 只会执行表中的"质问"或"告发"，而我们的 A 会根据性格说出："你这小人，竟背着同门干这等龌龊之事！"（如果 A 是刚烈正直的），或者"呵……有趣，倒不愧是姓李的。"（如果 A 是阴险冷淡的）。

**但前提是：PoC-2e 的准确率需要在实际游戏场景中验证。** 这又回到了我们之前审慎评估的结论——AI 目前是"有潜力的辅助"，不是"可替代规则的主力"。

### 4.4 实时+事后裁决（D2）

结合之前的讨论成果：
- **5 分钟黄金窗口**：重大事件发生前 → 系统给涉事弟子挂"对峙"状态 → 掌门如果在线可即时干预。
- **自动执行**：窗口过期 → AI/规则自动结算事件后果 → 时间线继续。
- **执法堂卷宗**：事件结果落入"待裁决"队列 → 掌门上线后可事后处理后果（包庇/惩罚/赏赐）。

## Part 5：来自 MUD 的最深刻启示

> **"世界感"从来不需要 3A 级别的技术。**

1996 年的《北大侠客行》用一堆 Room + 会走路的 NPC + 文字描述 = 一个让人沉迷十年的武侠世界。

我们已经有了：
- 8 个有性格、有道德、有关系的弟子（比 MUD 的 NPC 数据丰富 10 倍）
- 一个每秒心跳的 TickPipeline（MUD 的核心架构）
- 一台本地 AI（MUD 时代想都不敢想的能力）

我们缺的，仅仅是：
- 一个 `location` 字段（让弟子存在于"某个地方"）
- 一个事件池（让世界有"新鲜事"发生）
- 一套碰面规则（让平行线开始交叉）

**这不是一个需要几个月才能做完的庞大系统。它的 MVP 可能只是几天的工作量。**

## Part 6：USER 关键洞察记录（2026-03-29 21:02）

### 6.1 世界事件系统是"验证基础设施"而非"远期功能"

> **USER 原话**："我发现我就算改了一个新的系统上线，我甚至都没有办法验证这个系统是否已经生效了，这简直太扯淡了。"

**这是整个讨论中最重要的一句话。**

如果我们按 Roadmap 先做 Phase F（灵魂闭环：特性→行为权重、情绪→行为偏移、关系→行为权重），做完之后怎么验证？

- 验证"胆魄如虹的弟子更多选择 explore"——可以跑 Monte Carlo。✅
- 验证"刚烈正直的弟子在被偷丹药后会选择当面质问"——**没有"被偷丹药"这个事件！** ❌
- 验证"阴险毒辣的弟子在死敌虚弱时会趁机下手"——**不存在"死敌虚弱"这个场景！** ❌
- 验证 PoC-2e 的 AI 决策在"真实游戏场景"中的表现——**真实游戏根本产不出戏剧化场景！** ❌

**结论：世界事件系统不是 Phase I（远期）的锦上添花，而是 Phase F（近期）的验证前提。没有事件源，灵魂系统就是永远无法被测试的死代码。**

这直接影响 Roadmap 排序：世界事件系统（至少 MVP 级）必须在灵魂闭环之前或同时完成。

### 6.2 太吾绘卷的教训——复杂度是毒药

**太吾做对的：**
- 可以选正邪立场，各有妙处。不强迫玩家当好人。
- NPC 行为即事件源，因果链让世界真正"演化"。

**太吾做错的：**
- 世界推演系统（亲友/爱恨/强弱/NPC 关系）通过独立结算系统定期跑。
- **因为复杂度过高 + 作者代码能力和时代限制 → 系统频繁崩溃。**
- 作者花了**数年时间**重构优化，直接导致玩家大量流失。
- 新版本中大幅砍掉了世界推演系统 → 一切变得单调乏味。

**对我们的警告**：
> 不要试图一次性构建一个完美的世界模拟器。太吾的教训证明：过度复杂的周期性结算系统是工程炸弹。我们必须做到"最小复杂度产出最大戏剧效果"。

### 6.3 鬼谷八荒的教训——模式单一致命

**鬼谷做对的：**
- 技能系统有趣（初期）。
- 千里追杀等关系驱动事件有一定趣味。

**鬼谷做错的：**
- 缺乏主体故事线，世界线乏味。
- NPC 交互模式单一，重复感强 → 疲劳感来得极快 → 玩家退坑。
- 游戏后来的"复活"甚至依赖于第三方 Mod 的非核心玩法，而非系统本身的深度。

**对我们的启示**：
> 事件多样性和不可预测性是留存的核心。仅靠"千里追杀"一种因果事件是不够的。我们需要足够丰富（但不复杂）的事件池 + AI 生成的个性化叙事来对抗模式疲劳。

### 6.4 USER 对"正邪自由度"的隐含需求

> USER 原话："为何我的宗门不可以是魔宗？为何不可以是坏人堆？"

这揭示了一个我们之前完全没有讨论过的设计维度：**宗门的道德立场应该是可选的**。现有系统中 `moral.goodEvil` 只作用于个体弟子，但宗门整体的"正邪气质"——是行善积德的名门正派，还是弱肉强食的邪道魔宗——会根本性地改变事件的调性和玩家的体验。

这直接影响事件池的设计：不同立场的宗门应该有不同的事件倾向和可选行为。

## Part 7：待后续单开讨论的议题

以下议题每个都值得单独开篇深入探讨：

| # | 议题 | 为什么重要 |
|---|------|-----------|
| 1 | **事件密度与节奏**：多久发生一次、如何控制波峰波谷 | 直接决定游戏"手感"——太密则焦虑，太疏则无聊 |
| 2 | **事件严重度分级**：哪些自动执行、哪些等掌门裁决 | 决定玩家的参与感和放置自由度的平衡 |
| 3 | **Roadmap 重排**：世界事件 MVP 的定位和排期 | 影响整个开发优先级——必须在灵魂闭环之前或同时 |
| 4 | **宗门正邪立场系统** | 全新的设计维度，影响事件池、NPC 行为、玩家体验 |
