## §1 产品定位

### 1.1 一句话定义

> **用纯文字 MUD + AI 灵智弟子构建一个「每个弟子都有灵魂的活世界」。**

### 1.2 目标用户

| 维度 | 描述 |
|------|------|
| **核心画像** | 喜欢文字放置类游戏的修仙题材爱好者 |
| **平台** | 浏览器 Web（后期迁移 Electron） |
| **操作深度** | 低操作、高策略 — 弟子自动行动，宗主下达战略指令 |
| **使用场景** | 挂机为主、间歇查看 MUD 日志和弟子行为 |

### 1.3 核心假设

1. 纯文字 MUD + AI 灵智弟子能独立构成核心乐趣
2. 纯 Web（浏览器），后期移植 Electron
3. **AI 灵魂是核心差异化** - 弟子拥有五维性格、道德双轴、先天/后天特性、情感关系，AI 驱动个性化行为与对话
4. 骨架集 6 系统：修炼 + 弟子 + MUD + 灵田 + 炼丹 + 灵魂世界
5. 止步筑基圆满（验证范围最小化）
6. 弟子 8 人（4 初始 + 4 Phase D 新增），AI 事件驱动推理
7. **世界层是验证基础设施** - 没有事件源，灵魂系统无法被测试

### 1.4 成功指标

| 指标 | 目标 | 验证方式 |
|------|------|---------| 
| 核心循环可运行 | 从炼气1到筑基圆满全流程无阻断 | 自动化验证脚本 |
| AI 弟子有"生命感" | 弟子行为台词非千篇一律，性格差异可感知 | 人工体验 + Monte Carlo |
| 挂机体验流畅 | CPU ≤ 3%、内存 ≤ 200MB | 性能检测 |
| 系统闭环无通胀 | 灵石/灵气无净产出死循环 | 数值验证脚本 |
| **世界有"生命感"** | 15分内自然产生 ≥3 个有意义的事件 | 运行验证 |

## §4 系统清单与边界

### 4.1 IN/OUT 总表

| ✅ IN（lite 范围内） | 🚫 OUT（lite 范围外） |
|:---|:---|
| 修炼引擎（炼气 1 → 筑基圆满） | 金丹及以上境界 |
| 弟子行为树（8 人 × 7 态） | 弟子动态招募 / 离队 |
| AI 灵魂系统（性格/道德/特性/情绪/关系） | LoRA 微调 / 大模型切换 |
| NPC 世界层（地点/碰面/事件/道风/Storyteller） | 完整地图拓扑 / 3D 空间 |
| MUD 文字面板 + 命令输入 + 事件分级显示 | 图形渲染 / PixiJS / Canvas |
| 灵田种植（无限格，弟子×3块） | 废丹回收 / 宅邸灵气 |
| 炼丹（3 丹方） | 结丹天劫及以上 |
| 突破系统（概率模型） | 薪俸系统 |
| 宗门道风×门规双轴 + 掌门裁决 | 多宗门外交 |
| localStorage 存档 | 云存档 / 多设备同步 |
| 纯 Web 运行 | Electron 壳 |
