## L2: CoVe 证据验证

### WARN-1: `look <弟子>` 模糊匹配规则（R1-3）

**验证问题**：
1. 8 个弟子中是否有任何名字是另一个名字的前缀？
2. 如果输入"张"而有"张清风"和"张明远"两个结果怎么办？

**独立回答**：
1. 弟子由 `disciple-generator.ts` 随机生成，姓名池足够大，前缀冲突概率极低但不为零
2. 多结果场景需要处理——建议：列出所有匹配项让玩家选择

**判定**：⚠️ WARN 维持。建议在 PRD 中补充匹配规则：
- 先精确匹配全名 → 命中则直接显示
- 再前缀匹配 → 唯一命中则显示，多结果则列出候选
- 无匹配 → 提示未找到

### WARN-2: main.ts 膨胀风险（R5-5）

**验证问题**：
1. 当前 main.ts 多少行？新增命令后预估多少行？
2. 是否影响可维护性？

**独立回答**：
1. 当前 418 行 → 新增约 200~250 行（命令处理+渲染） ≈ 650 行
2. 650 行单文件可接受，但逼近重构阈值。不过 I3 约束了零引擎改动，拆分属于架构变更，应由 SGA 在 Phase G/H 主线中处理。

**判定**：⚠️ WARN 维持，记入技术债务，不阻塞本 Phase。
