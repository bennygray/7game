### 1a. 5-Why 根因链

```
为什么需要道德/特性/关系系统？
  → 因为当前弟子只是「穿了不同衣服的同一个灵魂」
    → 为什么这是问题？因为行为树用连续五维值（aggressive/persistent/kind/lazy/smart）
      做加权随机，差异仅体现在行为概率分布微调，NPC 感知不到个性
      → 为什么加权随机不够？因为它无法产生「戏剧性」——所有弟子面对同一事件
        的反应模式是无差异的概率分布
        → 为什么需要戏剧性？因为太吾/鬼谷的核心乐趣不是「NPC 做了什么」
          而是「NPC 为什么做了这件事」——动机可解释性才是灵魂
          → 根因价值：【可解释的差异化动机】= NPC 的灵魂
```

> **根因价值**：让每个弟子拥有**可解释的行为动机**，使玩家能理解「为什么张清风选择背叛而陈明月选择原谅」——这种可解释性才是「血肉」和「灵魂」的本质。

### 1b. Invariant（不变量）

| # | 不变量 | 违反后果 |
|---|--------|---------|
| I1 | **游戏引擎（规则层）永远是最终仲裁者** — AI 只能在规则允许的范围内影响行为 | AI 幻觉导致游戏崩溃 |
| I2 | **AI 不可用时游戏功能 100% 正常** — 所有 AI 增强必须有确定性 fallback | 游戏卡死 |
| I3 | **NPC 状态变更必须经过规则验证** — AI 不能直接修改 GameState | 数值被污染 |
| I4 | **小模型的输出必须是结构化的** — 自由文本仅用于台词展示 | 解析失败、行为不可控 |
| I5 | **道德/特性值的变化幅度有硬上限** — 防止模型输出极端值导致状态突变 | 人物性格一夜剧变 |
| I6 | **关系值是双向独立的** — A→B 和 B→A 可以不对称（参考鬼谷八荒） | 关系不真实 |

### 1c. 最小组件检验

> 如果只保留一个核心功能：**AI 事件评估器**。
>
> 理由：当一个事件发生（如「陈明月炼丹成功」），AI 评估每个相关弟子的内心反应，
> 输出结构化 JSON（情绪标签 + 关系变化 + 台词），而非仅生成台词。
> 这是从「AI 念台词」到「AI 有灵魂」的最小跳跃。

### 1d. 核心体验锚定

| 维度 | 内容 |
|------|------|
| **核心体验** | 观察弟子在事件中展现出**符合其道德/特性的差异化反应**，产生「这个人果然是这样的性格」的会心一笑 |
| **ROI** | L 成本（预研 Phase，含大量实验）/ 体验 5 分（成功的话是质变） |
| **循环挂载** | 事件发生 → AI 评估反应 → 关系变化 → 影响下一次行为决策 → 产生新事件 → ... |

## 关键调研发现

### 太吾绘卷/鬼谷八荒的设计哲学

| 游戏 | 核心机制 | 我们可以借鉴的 | 我们必须超越的 |
|------|---------|--------------|--------------|
| **太吾绘卷** | 处世立场（仁善/中庸/叛逆/唯我/刚正）× 七元赋性 → 奇遇决策路径 | 离散立场标签 + 连续数值的混合模型 | 太吾的 NPC 行为是硬编码 IF-ELSE 驱动 |
| **鬼谷八荒** | 性格词条 × 好感度月结衰减 × 正魔立场 | 好感度衰减机制 + 性格影响交互偏好 | 鬼谷的行为本质是查表法 |
| **两者共同点** | 简单规则 × 大量 NPC = 涌现性复杂度 | ✅ 我们的 8 弟子可以靠 AI 弥补数量不足 | NPC 行为的「机械感」和「可预测性」 |

### Stanford Generative Agents 论文启示

| 概念 | 原论文 | 我们的适配方案 |
|------|--------|-------------|
| **Memory Stream** | 存储所有经历 + 反思 | ⚠️ 0.8B 模型 context 有限 → 精简为 **事件摘要队列** |
| **Reflection** | 定期从记忆中生成高级推理 | ✅ 可做 → 用 AI 定期「反思」最近事件，更新目标/情绪 |
| **Planning** | 生成每日计划 | ⚠️ 0.8B 能力不足 → plan 由规则引擎生成，AI 只做「偏好调整」 |
| **Retrieval** | 按 Recency/Importance/Relevance 检索 | ✅ 用简单的时间衰减 + 重要性标签即可（无需向量数据库） |

### 0.8B/4B 小模型能力边界（关键发现）

> [!IMPORTANT]
> **0.8B 模型的核心能力**：分类、情感分析、短文本生成、模板填充、简单 JSON 输出
> **0.8B 模型的核心弱点**：复杂推理、长上下文理解、多步规划、角色一致性维持

**可行的任务（0.8B 实测有信心）**：
1. 给定事件描述 + NPC 性格档案 → 输出情绪标签（1 of N 分类）
2. 给定事件 + 情绪 → 输出 1-2 句台词（当前已验证）
3. 给定结构化 prompt → 输出简单 JSON（`{emotion, reaction, line}`）

**高风险任务（需 4B 或更优化的 prompt）**：
1. 多人社交推理（A 和 B 吵架了，C 应该站哪边？）
2. 长期性格一致性（跨 100+ 事件保持同一角色设定）
3. 复杂叙事生成（创造剧情转折）

## 核心架构提案：「三脑协同」模型

> [!IMPORTANT]
> 这不是推翻现有架构，而是在现有「左右脑」基础上增加**第三个脑** — AI 评估器。

```
┌─────────────────────────────────────────────────────────┐
│                    Event Bus（事件总线）                   │
│   "陈明月炼丹成功"  "张清风修炼达到瓶颈"  "周星河又偷懒了"    │
└────────┬──────────────────┬──────────────────┬──────────┘
         ▼                  ▼                  ▼
┌──────────────┐  ┌──────────────────┐  ┌──────────────────┐
│  脑 1: 规则层  │  │ 脑 2: AI 评估器   │  │  脑 3: AI 叙事器  │
│ （确定性引擎）  │  │（结构化 JSON 输出）│  │ （自由文本台词）  │
│              │  │                  │  │                  │
│ • 行为树权重  │  │ • 情绪分类       │  │ • 对话生成       │
│ • 关系规则    │  │ • 关系 delta 建议  │  │ • 旁白/独白      │
│ • 特性触发    │  │ • 特性触发判定    │  │ • 内心活动       │
│ • 数值计算    │  │ • 行为偏好微调    │  │                  │
│              │  │                  │  │                  │
│ 输出: 确定值  │  │ 输出: JSON       │  │ 输出: 文本       │
│ 无需AI       │  │ 0.8B 可做        │  │ 0.8B 已验证     │
└──────┬───────┘  └────────┬─────────┘  └────────┬─────────┘
       │                   │ (需规则验证)          │
       ▼                   ▼                      ▼
┌─────────────────────────────────────────────────────────┐
│              State Validator（状态验证层）                │
│   AI 建议的关系变化经过规则引擎裁剪（夹值、冷却、上限）       │
└─────────────────────────┬───────────────────────────────┘
                          ▼
                    GameState 更新
```

### 关键设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| AI 的角色 | **建议者**，不是决策者 | I1 不可违反；规则引擎有最终否决权 |
| 评估频率 | **事件驱动**，非每 tick | 0.8B 推理速度限制 + 8 弟子并发 |
| JSON 或自由文本 | **JSON 用于状态变化，自由文本仅用于展示** | I4 结构化输出是小模型可靠性的关键 |
| 关系变化来源 | **规则层 70% + AI 建议层 30%** | 确定性基线 + AI 锦上添花 |
| Fallback 策略 | **规则引擎独立运行，AI 增强为可选层** | I2 完整 fallback |

## Step 2：业务实体与规则

### 2.1 新增业务实体

#### 道德系统（双轴）

```typescript
/** 道德轴 — 善恶 × 正邪 */
interface MoralAlignment {
  /** 善恶轴: -100(极恶) ~ +100(极善) */
  goodEvil: number;
  /** 正邪轴: -100(邪道) ~ +100(正道) */
  lawChaos: number;
}
```

#### 特性系统

```typescript
/** 特性定义 */
interface TraitDef {
  id: string;                    // 'alchemy-prodigy', 'greedy', 'flirtatious'
  name: string;                  // '丹道圣体', '爱财如命', '水性杨花'
  category: 'innate' | 'acquired';  // 先天 / 后天
  polarity: 'positive' | 'negative' | 'neutral';
  effects: TraitEffect[];       // 数值效果
  aiHint: string;               // 给 AI 的性格提示词
}

/** 特性效果 */
interface TraitEffect {
  type: 'behavior-weight'       // 行为权重修正
    | 'alchemy-bonus'           // 炼丹加成
    | 'relationship-modifier'   // 关系变化倍率
    | 'aura-multiplier'         // 灵气产出倍率
    | 'moral-drift';            // 道德倾向漂移
  target: string;               // 目标行为/指标
  value: number;                // 修正值
}

/** 弟子持有的特性实例 */
interface DiscipleTrait {
  defId: string;                // 指向 TraitDef.id
  acquiredAt: number;           // 获得时间戳
  source: string;               // 来源（'innate' | 事件描述）
}
```

#### 关系系统（升级）

```typescript
/** 关系边（升级版，替代现有 RelationshipEdge） */
interface RelationshipEdge {
  sourceId: string;
  targetId: string;
  /** 好感度 -100 ~ +100 */
  affinity: number;
  /** 关系标签（可叠加） */
  tags: RelationshipTag[];
  /** 最后互动时间（用于衰减） */
  lastInteraction: number;
}

type RelationshipTag = 'friend' | 'rival' | 'mentor' | 'admirer' | 'grudge';
```

#### AI 事件评估（核心新增）

```typescript
/** AI 评估请求 */
interface SoulEvaluationRequest {
  event: GameEvent;              // 发生的事件
  subject: DiscipleProfile;     // 产生反应的弟子
  involved: DiscipleProfile[];  // 涉及的其他弟子
}

/** AI 评估结果（结构化 JSON） */
interface SoulEvaluationResult {
  emotion: EmotionTag;           // 情绪分类
  intensity: 1 | 2 | 3;         // 强度 1-3
  relationshipDeltas: Array<{    // 关系变化建议
    targetId: string;
    delta: number;               // -10 ~ +10（规则引擎会裁剪）
    reason: string;              // 原因（展示用）
  }>;
  behaviorBias?: {               // 下一次行为偏好（可选）
    behavior: string;
    weight: number;              // 0.5 ~ 2.0 倍权重修正
  };
  innerThought: string;          // 内心独白（展示用）
}

type EmotionTag = 'joy' | 'anger' | 'envy' | 'admiration'
  | 'sadness' | 'fear' | 'contempt' | 'neutral';
```

## PoC 验证计划（本 Phase 核心）

> [!IMPORTANT]
> 作为预研型 Phase，以下 PoC 实验的结论将决定正式实施方案。

### PoC-1：AI 结构化 JSON 输出能力（⚠️ 最高优先级）

**目标**：验证 Qwen3.5-0.8B 能否可靠输出结构化 `SoulEvaluationResult`

**方法**：
1. 构造 20 个典型事件场景（炼丹成功/修炼瓶颈/偷懒被发现/...）
2. 为每个场景搭配 3 种不同性格的弟子
3. 用 llama.cpp Grammar 约束 JSON schema
4. 记录：成功率、延迟、输出质量评分

**通过标准**：
- JSON 格式正确率 ≥ 80%（有 Grammar 约束下）
- P95 延迟 ≤ 3s
- 情绪分类与场景匹配度 ≥ 70%（人工评分）

**失败对策**：如 0.8B 不够 → 测试 4B；如 4B 仍不够 → 降级为「规则分类 + AI 台词」

### PoC-2：AI 驱动的关系变化（中优先级）

**目标**：验证 AI 建议的关系 delta 是否合理

**方法**：
1. 10 个事件 × 3 对弟子组合 = 30 个 case
2. AI 输出 relationship delta + reason
3. 对比规则引擎的确定性计算
4. 人工评分合理性

**通过标准**：
- AI 建议与人类直觉一致率 ≥ 60%
- 无极端异常值（delta 超出 [-10, +10]）

### PoC-3：事件→反应→关系→行为 闭环（低优先级，PoC-1/2 通过后）

**目标**：验证完整循环是否产生有趣的涌现行为

**方法**：
1. 让 8 弟子运行 5 分钟（≈100+ 事件）
2. 追踪关系图变化
3. 观察是否出现「戏剧性」事件（如两弟子因竞争从友变敌）

## USER 决策点

> [!IMPORTANT]
> 以下问题需要你决策后才能进入 Step 3（User Story）。

### Q1：道德初始分配方式 — ✅ 已决策

**USER 决策**：**道德与性格是独立维度，组合关系而非绑定关系。**

> 用户原话摘要：
> - 存在「性格确定理论」— 性格和道德不是绑定的，是组合关系
> - 道德可以有微小变化：道德接近的人容易相处，相处久了道德会趋同
> - 性格完全不同的人可以互补，但改变性格是极难的
> - 所以道德是可变的（受关系影响），性格是近乎不可变的

**设计含义**：
- 道德双轴值：**低可变性**（初始随机 ± 特性微调，受关系驱动缓慢漂移）
- 性格五维值：**几乎不可变**（生成时确定，仅极端事件可能微调）
- 关系→道德反馈：长期友好关系的弟子，道德值缓慢趋同（新规则 R-E14）
- 性格互补：不同性格可以产生正面关系（如「刚烈」×「温和」= 互补吸引）

**R-E14 道德趋同规则（新增）**：
- 触发条件：两弟子 affinity ≥ 50 且持续 > 5 分钟
- 效果：双方道德值每分钟向对方漂移 0.1（夹值到初始值 ± 20 范围内）
- 约束：不改变道德区间标签（善→恶的跨区漂移需要重大事件）

### Q2：PoC 优先级确认 — ✅ 已决策

**USER 决策**：**PoC-1 作为第一步。接受降级方案作为兜底。**

> - PoC-1（JSON 输出能力）优先执行
> - 如不理想，可做模型级别调整（切换 4B 或其他模型）
> - 「规则引擎 + AI 台词」作为最终退路也可接受（没有办法的办法）

**执行策略**：PoC-1 → 评估 → 通过则继续 / 不通过则 4B 重测 → 仍不通过则降级为规则+台词

### Q3：特性池的设计方向 — ✅ 已决策

**USER 决策**：**B) 人性混合。修仙是壳，人性是核。**

> 用户原话摘要：
> - 修仙只是最容易被接受的壳
> - 本质是人性混合 — NPC 的关系行为和自成故事体系，比修仙外壳吸引人得多
> - 如果底层逻辑能实现，换个壳也完全可以

**设计含义**：
- 特性池优先设计人性维度（贪婪、忠诚、嫉妒心强、爱冒险...）
- 修仙只是包装（丹道圣体 = 对应某种人性特质的修仙语境表达）
- 特性系统的核心价值是**驱动可解释的行为差异和涌现叙事**

### Q4：4B 模型作为升级路径 — ✅ 已决策

**USER 决策**：**不拒绝任何尝试。**

- 0.8B → 4B → 更大模型，渐进升级路径完全开放
- 推理延迟和显存成本是可接受的代价
