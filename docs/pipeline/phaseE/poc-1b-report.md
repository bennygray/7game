## 关键发现

### ✅ 完全通过的方面

1. **JSON 格式可靠性 100%** — 即使 prompt 复杂度翻了 10 倍，response_format json_schema 约束仍然完美工作
2. **Schema 完整率 100%** — 所有必填字段（emotion, intensity, deltas, innerThought）均正确输出
3. **GPU 加速效果显著** — 平均 ~660ms，P95 ~960ms，完全在可接受范围内
4. **关系 Delta 方向 70% 合理** — 大多数场景中的好感变化方向正确

### ⚠️ 需要改进的方面

1. **情绪分类准确率 55%** — 主要问题在「jealousy 泛化」：
   - R01: 陈明月发现追求者 → 输出 jealousy（期望 joy/worry/neutral）
   - R05: 好友为保护自己受伤 → 输出 jealousy（期望 sadness/guilt/admiration）
   - R13: 追求者照顾受伤的自己 → 输出 envy（期望 admiration/guilt）
   - R16: 好友为救门派重伤 → 输出 jealousy（期望 sadness/anger/worry）
   - R17: 弟子叛逃 → 输出 pride/contempt（期望 anger/sadness）
   - R18: 死敌救命 → 输出 anger/contempt（期望 admiration/guilt）

2. **Delta 正负方向问题** — 6 个场景的 delta 方向异常

### 💡 根因分析

0.8B 模型的核心弱点是 **情绪细粒度区分能力不足**：
- 容易把「感动/愧疚/担忧」统一归类为 jealousy/envy
- 在涉及复杂社交动态时，情绪判断退化为最简单的负面分类
- 但在「明确的」场景（偷盗→anger、竞争失败→anger、背叛→anger）中表现优秀

## 结论与建议

### 结构化输出能力：✅ 完全可用
- JSON + Schema 100% 可靠，延迟 <1s

### 情绪判断能力：⚠️ 需规则层辅助
**建议方案：规则引擎情绪候选池**
- 规则层根据事件类型 + 关系 + 道德 → 预设「情绪候选池」（3-4 个）
- AI 只在候选池内做 N 选 1 分类
- 0.8B 的分类能力在受限选择下足够可靠

### 升级路径
1. 实现「情绪候选池」规则层 → 重跑 PoC-1b
2. 如候选池方案仍不够 → 切换 4B 模型重测
3. 如都不行 → 降级为规则引擎 + AI 台词
