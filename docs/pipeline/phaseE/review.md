> **信条**：每一个数字都必须有公式出处，每一条产出都必须有消耗兜底。

| 维度 | 判定 | 说明 |
|------|:----:|------|
| D1 漏斗平衡 | ✅ | 道德（善恶/正邪）、关系（affinity）和特性（traits）均为状态描述值，不参与资源经济循环，无漏斗平衡问题。对照 `prd/economy.md` 验证：灵魂系统未引入新的可消耗/可积累资源。特性有上限 cap（2 先天 + 3 后天 = 5），天然防止积累膨胀。 |

| D4 Sink完备 | ✅ | 无新资源引入。道德/关系是状态描述值，不参与 `prd/economy.md` 的资源漏斗。特性 cap=5 天然 Sink。灵魂系统与灵气/灵石/丹药经济完全隔离（R-E13 关系影响行为权重是概率修正 ×1.3，不改变资源产出量）。 |
| D5 二阶效应 | ✅ | R-E13 `friend → 合作行为权重 ×1.3` 是加法修正概率权重（影响行为选择概率），不是乘法修正资源产出量。与现有修速丹(×2)、灵脉密度(×1.0~5.0)、道基(×1.5~8.0) 的乘法链不交叉。验证：即使 8 弟子全部 friend 关系 → 修炼选择概率从 25% → 32.5%，产出不变，无数值爆炸路径。 |

### W2：R3-D2 — 24h 衰减趋零问题

**原始 WARN**：0.98^288 ≈ 0.003，24h 挂机后关系趋零

**CoVe 验证问题**：
1. 0.98^288 实际等于多少？
2. 鬼谷八荒如何处理长期衰减？
3. 是否需要衰减下限？

**独立回答**：
1. `0.98^288 = e^(288 × ln(0.98)) = e^(288 × (-0.0202)) = e^(-5.82) ≈ 0.003`。确认趋零。
2. 鬼谷八荒：月结衰减有绝对值下限，好感度不降到指定阈值以下。且有「记忆衰减」概念但保留「印象」。
3. 需要。建议增加 R-E15：|affinity| ≤ 5 时停止衰减。这保证有意义的关系历史不被完全清除。

**判定**：**维持 WARN → 修复方案**：新增 **R-E15 衰减下限**（|affinity| ≤ 5 时停止衰减）。此规则已追加到 PRD §3.3。
- 验证：affinity=80, 24h 后 → max(80×0.98^288, 5) = max(0.24, 5) = **5**。✅ 不趋零。
- 验证：affinity=3, 任意时间 → 不衰减（已在阈值内）。✅ 微弱关系稳定。
