## L2：CoVe 证据验证

### R2-D3 WARN — 系统消息条折叠后灵气增长不可见

**验证问题**：
1. 当前状态栏是否已显示灵气增速？  
2. 折叠系统消息条后灵气信息是否完全消失？

**独立回答**：
1. ✅ 是，状态栏 `formatStatusBar` 已显示 `灵气：xxx (+12.0/s)`，实时更新
2. ❌ 不会消失。灵气增速在状态栏实时可见，系统消息条仅显示"打坐入定，灵气+60"等冗余文案

**CoVe 结论**：WARN 降级为 **PASS** — 状态栏已覆盖灵气信息，系统消息条的修炼日志确实是冗余信息。

### R5-D4 WARN — formatter 与 CSS 隐式耦合

**验证问题**：
1. 有多少个 severity class 需要同步？
2. 是否有先例（行业实践）证明这是可接受的？

**独立回答**：
1. 5 个（BREATH/RIPPLE/SPLASH/STORM/CALAMITY），规模很小
2. ✅ 这是 CSS 框架的标准模式（Bootstrap/Tailwind 都是 class-based），是行业最佳实践

**CoVe 结论**：维持 **WARN** 但标注为低风险 — 建议 CSS class 使用 `mud-` 前缀命名约定即可。
