## 3. 质量指标（修正后）

### 3.1 幻觉检测修正说明

> **原始检测器存在严重的假阳性问题**。
>
> 旧检测逻辑：标点后 3 个中文字 + "的/在/是/又…" → 提取为疑似人名 → 若不在白名单则判定为幻觉。
>
> 问题：大量正常中文短语被误判为"虚构人名"：
> - `"心中满是欢喜"` → `心中满` → 误判 ❌
> - `"哼，又是这个死对头"` → `哼，又` → 误判 ❌
> - `"居然敢在我面前"` → `居然敢` → 误判 ❌
>
> 2B 模型因使用更口语化的表达风格，触发了更多假阳性（30% vs 17%），造成了"2B 幻觉率更高"的错误结论。
>
> **修正方案**:
> 1. 新增 80+ 条常见中文短语白名单
> 2. 新增姓氏首字过滤（只有首字为百家姓的 3 字词才视为疑似人名）
> 3. 保留 prompt 包含性检查
>
> **修正结果**: 三个模型的幻觉率均降至 **0%**，即在当前测试中没有检出真正的虚构人名。

## 附录 A: 上下文级别影响模型

```
Context Level 对模型行为的影响路径:

L0 (无信息)     → 模型依赖 "世界知识" 猜测关系 → 2B 更优（知识更丰富）
L1 (标签+好感)  → 模型获得关系方向 → 2B 情绪识别显著提升
L2 (1条经历)    → 模型获得具体事件 → 两者都大幅提升，2B 综合分最高 (93)
L3 (3条经历)    → 信息饱和 → 综合分稳定，但 2B 出现 PROTECT bias
L4 (+个人近况)  → 自我认知强化 → 各模型表现趋同
L5 (+间接关系)  → 社交网络信息 → 未显著改善决策
L6 (+叙事片段)  → 叙事性强化 → 2B T4 正确率从 20% 跳升至 80% ★
```

## 附录 B: PROTECT Bias 现象解释

2B 模型在 L3-L5 的 T4 场景中出现显著的 PROTECT bias（面对死对头被困也倾向保护），原因分析：

1. **RLHF 训练偏差**: 大模型经过更多的人类反馈对齐，习得了"救人优先"的道德默认值
2. **信息不充分**: L3 只列举了 3 条冲突事件（羞辱、争夺、暗算），缺乏"为什么不值得救"的归纳性叙事
3. **L6 的解决机制**: 叙事片段 `"李沐阳趁其闭关造谣，暗中拉拢弟子"` 将冲突从"事件层面"升级为"人格层面"的恶意，成功覆盖了 RLHF 的善意默认值

这一发现对 prompt engineering 的启示：  
**在需要 NPC 表现出非善意行为的场景中，必须提供叙事性上下文（narrative snippet）而非仅事件列表。**
