# Phase H-γ — 掌门裁决 PRD

> **版本**：v1.0 | **日期**：2026-03-31 | **作者**：/SPM
> **Phase 定位**：H 剩余（H4 干预窗口 + H6 裁决→道风漂移），H5 执法堂拆到下一迭代
> **核心假设**：「玩家裁决 → 道风漂移 → 世界调性变化」闭环可验证

---

## §1 需求概述

### 1.1 问题陈述

Phase H-β 交付后，玩家能通过 MUD 看到完整世界（碰面/事件/独白/档案/道风），但宗门道风（ethos/discipline）自开局设置后**从未被玩家操作改变过**。Lv.3 STORM 事件虽已触发，但自动结算后无任何后果——掌门角色是空壳。

### 1.2 价值锚定

| 维度 | 内容 |
|------|------|
| 核心体验 | "我是掌门——Lv.3 风暴事件由我裁决，道风因我的选择而改变" |
| ROI | 成本 M · 体验增量 5/5（从旁观者到掌门的质变） |
| 循环挂载 | STORM → 裁决窗口 → 选项 → 道风漂移 → 影响事件池权重 → 新 STORM |
| 需求债务清偿 | FB-011（玩家干预权缺失） |

### 1.3 范围边界

| IN | OUT |
|:---|:----|
| Lv.3 STORM 世界事件的即时裁决 | Lv.4 CALAMITY 事件裁决（Phase I） |
| MUD 命令式裁决（`judge 1`/`judge 2`） | 弹窗式 UI / HTML 浮层 |
| 裁决→道风漂移（±1~2） | 裁决→弟子属性变更（需 action-executor） |
| 超时自动结算 | 执法堂卷宗系统（H5，下一迭代） |
| encounter-conflict 排除 | encounter-conflict 裁决（后续迭代） |

## §2 不变量

| # | 不变量 | 违反后果 |
|---|--------|---------|
| I1 | 干预窗口必须有超时自动结算 | 事件永远挂起，阻塞世界推进 |
| I2 | 裁决选项必须 ≥2 且互斥 | 无真正的选择 |
| I3 | 每次裁决的道风漂移幅度 ≤2 | 道风剧烈震荡，破坏渐进感 |
| I4 | 错过干预不惩罚玩家 | 放置/挂机玩家体验崩塌 |
| I5 | 裁决数据只读 GameState，不修改弟子属性 | 越权修改导致未知副作用 |
| I6 | UI 层不直接修改 GameState | 架构红线 X-1 |

## §3 业务实体

### 3.1 RulingDef — 裁决定义（静态注册表条目）

每个可触发 STORM 的世界事件关联一组裁决定义。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 裁决唯一 ID，格式 `RD-{eventId}-{序号}` |
| eventDefId | string | 关联的 WorldEventDef.id |
| label | string | 选项标签（中文，≤20字） |
| description | string | 选项描述（中文，1~2 句话） |
| ethosShift | number | 道风漂移值（-2~+2，负=偏仁，正=偏霸） |
| disciplineShift | number | 门规漂移值（-2~+2，负=偏放，正=偏律） |
| mudText | string | 裁决后的 MUD 输出文案 |

### 3.2 ActiveRuling — 活跃裁决（运行时状态）

当 STORM 事件触发时创建，超时或玩家裁决后销毁。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 运行时唯一 ID |
| eventPayload | WorldEventPayload | 触发此裁决的事件载荷 |
| options | RulingOption[] | 可选裁决项（从 RulingDef 实例化） |
| createdAt | number | 创建时间戳（inGameWorldTime） |
| expiresAt | number | 过期时间戳 |
| resolved | boolean | 是否已裁决 |
| chosenOptionIndex | number \| null | 玩家选择的选项索引（null=超时） |

### 3.3 RulingOption — 裁决选项（运行时实例）

| 字段 | 类型 | 说明 |
|------|------|------|
| index | number | 选项序号（1-based，用于 `judge N` 命令） |
| label | string | 选项标签 |
| description | string | 选项描述 |
| ethosShift | number | 道风漂移值 |
| disciplineShift | number | 门规漂移值 |

## §4 规则与数值

### R-01 裁决窗口触发规则

| 条件 | 值 | 说明 |
|------|------|------|
| 触发条件 | WorldEventPayload.severity === STORM (3) | 仅 Lv.3 风暴级 |
| 排除条件 | 当前已有 ActiveRuling 未结算 | 同时只允许 1 个裁决窗口 |
| 窗口时长 | 300 秒（5 分钟） | 与 STORM_COOLDOWN_S 一致 |

### R-02 裁决选项来源

每个 STORM 级世界事件在裁决注册表中预定义 2~3 个裁决选项。当前 STORM 事件仅 WE-B04（妖潮来袭），但任何事件经五级漏斗升级到 STORM 也会触发裁决。

**裁决选项匹配逻辑**：
1. 优先查找 `RulingDef.eventDefId === payload.eventDefId` 的专属选项
2. 若无专属选项（事件被漏斗升级，原始定义无裁决），使用 **通用裁决选项池**（按 polarity 分组）

### R-04 超时自动结算

| 规则 | 值 |
|------|------|
| 超时时长 | 300 秒（5 分钟） |
| 超时行为 | 随机选择一个选项（等概率） |
| 超时 MUD 提示 | "掌门未及时裁决，事态自行发展……" + 被选选项的 mudText |
| 道风漂移 | 按被选选项的 ethosShift/disciplineShift 执行 |

**设计理由（I4）**：错过不惩罚。随机选择而非"最差选项"，挂机玩家不会被持续惩罚。

### R-05 道风漂移执行

| 规则 | 值 |
|------|------|
| 漂移公式 | `sect.ethos = clamp(sect.ethos + option.ethosShift, -100, 100)` |
| | `sect.discipline = clamp(sect.discipline + option.disciplineShift, -100, 100)` |
| 漂移上限 | 单次 \|ethosShift\| ≤ 2，\|disciplineShift\| ≤ 2 |
| 持久化 | 随 GameState 自动存档 |

### R-06 MUD 交互协议

#### 裁决窗口开启时的 MUD 输出

```
═══════════════════════════════════════
⚡ 【风暴事件】{事件名}
{事件文案（已渲染弟子名的 template）}

请掌门裁决：
  [1] {选项1.label} — {选项1.description}
  [2] {选项2.label} — {选项2.description}
  [3] {选项3.label} — {选项3.description}（如有）

输入 judge 1/2/3 做出裁决（剩余 {秒数} 秒）
═══════════════════════════════════════
```

#### `judge N` 命令

| 条件 | 行为 |
|------|------|
| 无活跃裁决 | 输出 "当前没有待裁决的事件。" |
| N 超出范围 | 输出 "无效选项，请输入 judge 1~{max}。" |
| 正常裁决 | 执行道风漂移 + 输出裁决 mudText + 清除 ActiveRuling |

#### `judge` 命令（无参数）

显示当前活跃裁决的详情和剩余时间，等同于重新打印裁决窗口。无活跃裁决时提示"当前没有待裁决的事件。"

### R-07 裁决窗口与 Storyteller 的时序关系

```
Storyteller.tick() → STORM payload
  ↓
world-event-tick.handler → emit SoulEvent + 日志
  ↓
裁决系统检查：severity === STORM && 无活跃裁决？
  → 创建 ActiveRuling → MUD 输出裁决窗口
  ↓
300 秒内：
  - 每 tick 检查超时
  - 玩家 `judge N` 命令 → 结算
  ↓
结算（玩家裁决或超时）：
  → 道风漂移 → mudText 输出 → 清除 ActiveRuling
  ↓
Storyteller 喘息期继续（STORM_COOLDOWN_S 不受裁决影响）
```

## §6 持久化考量

| 数据 | 持久化 | 说明 |
|------|:------:|------|
| RulingDef 注册表 | 否 | 静态代码，Object.freeze |
| ActiveRuling | 否 | 运行时状态，页面刷新重置（与 Storyteller 一致，ADR-F0β-01） |
| sect.ethos/discipline | 是 | 已在 GameState 中，随存档持久化 |

**设计理由**：ActiveRuling 不持久化与 StorytellerState 保持一致。页面刷新后裁决窗口消失，Storyteller 重置，不影响玩家体验（挂机场景本就会错过大部分裁决）。

## §8 效果数值表

| 选项类型 | ethosShift 范围 | disciplineShift 范围 | 说明 |
|---------|:---------:|:---------------:|------|
| 仁道倾向 | -1 ~ -2 | 0 ~ +1 | 慈悲/保护/共享 |
| 霸道倾向 | +1 ~ +2 | 0 ~ -1 | 竞争/独享/历练 |
| 律法倾向 | 0 | +1 ~ +2 | 规矩/严查/布阵 |
| 放任倾向 | 0 | -1 ~ -2 | 顺其自然/放手 |

**渐进感验证**：以最高漂移 ±2/次计，STORM 频率 2~4/h，1 小时最大漂移 ±8。从中立 (0) 滑到象限边界 (±40) 需约 5~10 小时——符合"水滴石穿"设计意图（09-sect-alignment.md §3）。
